# Spec: Image Game

## Goal
Teams identify people, objects, or places from images; questions are driven by files in a filesystem directory rather than JSON config.

## Acceptance criteria
- [x] Questions are auto-generated server-side by scanning the `image-guess/` directory
- [x] Each image file becomes one question; the filename (without extension) is the answer
- [x] Files prefixed `Beispiel_` are treated as example questions and appear first
- [x] Remaining questions are randomised on each server request
- [x] Image is shown to teams via the projected display; host reveals the answer text on keypress
- [x] Images are zoomable via a `Lightbox` overlay (click on image to zoom)
- [x] After the last image, calls `onGameComplete()`
- [x] Supported formats: common image types (`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`)

## State / data changes
- No `AppState` changes
- Config type: `ImageGameConfig` in `src/types/config.ts`
- Questions generated by server in `server/index.ts`
- Served from static path: `/image-guess/<filename>`

## UI behaviour
- Component: `src/components/games/ImageGame.tsx`
- Full-width image displayed; click opens `Lightbox` for zoom
- Answer hidden until host presses Space/click; answer text shown below or overlay
- `Lightbox` component: `src/components/layout/Lightbox.tsx`

## Out of scope
- Cropped/partial image reveals
- Multiple images per question
- Manual question entry in JSON (always filesystem-driven)
